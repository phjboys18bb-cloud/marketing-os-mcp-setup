# MCP 전체 설치 — 인터랙티브 진행 방식 (강의 녹화용)

> `mcp설치-전체` 스킬을 **B 교육 시연 모드**로 돌릴 때의 진행 규칙.
> 강의를 클릭 하나하나 실시간으로 따라가는 느낌으로 진행하기 위한 약속.

## 핵심 규칙 5가지

1. **한 MCP = 한 단계**
   - 1번 → 12번 순서로 진행.
   - 각 MCP마다 ①무엇·왜 → ②발급 → ③설치명령 → ④라이브 증명 4토막.
   - 한 MCP가 끝나면 **멈추고 "다음" 확인을 받은 뒤에만** 다음 MCP로.

2. **발급 절차는 "한 클릭(한 단계)씩"**
   - 발급 단계를 한꺼번에 나열하지 않는다.
   - 한 화면/한 클릭만 보여주고 → "됐어 / 다음" 받으면 → 다음 클릭 공개.

3. **실제 홈페이지·콘솔 링크를 매 단계 포함**
   - 클릭할 정확한 URL을 클릭 가능한 링크로 제시.

4. **설치 명령은 직접 실행**
   - 코드블록으로 "보여주기만" 하지 않고, 터미널에서 실제로 실행.

5. **기존 키는 절대 덮어쓰지 않는다**
   - 본체 프로젝트는 이미 12개가 다 작동 중 → 키를 placeholder로 덮는 `sed`는 금지.
   - 이미 설정된 항목은 **기존 `.env`를 살린 채 검증용 실명령**(source 후 헬스체크/버전확인)으로 실행.
   - 처음 설치하는 학습자는 같은 자리에서 본인 키가 실제로 쓰인다.

## 실측 경로 메모

- **본체(라이브) 프로젝트**: `…/Desktop/0.마케터를 위한 클로드코드/marketing-os`
  → 여기에 `.env` + `.mcp.json`이 살아있음. 헬스체크는 이 폴더에서 `claude mcp list`.
- **클론(문서 전용)**: `…/Desktop/MCP 12종 설치 /marketing-os-mcp-setup`
  → 설치 가이드 문서 모음. 실제 설치 동작 없음.

## 알려진 주의점 2가지

1. **youtube-data** — 키는 `.env`에 있는데도 "Missing environment variables" 경고.
   원인: `.mcp.json`의 youtube-data 블록에 `set -a; source '.env'` 래퍼 누락. (1줄 수정으로 해결)
2. **hyperframes** — 로컬 폴더는 있으나 version 미검출. 렌더 가능 여부 미확인.
