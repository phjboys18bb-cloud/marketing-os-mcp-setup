## 1. 시작 전 30초 — 설치 3가지 패턴만 기억하면 끝

영상에 나오는 모든 설치는 아래 **3가지 패턴 중 하나**입니다. 이것만 알면 12개가 다 똑같이 보입니다.

| 패턴 | 어떻게 | 해당 MCP |
|---|---|---|
| **A. Claude.ai 통합 (1클릭)** | Claude.ai > Settings > Integrations > Connect | Figma · Higgsfield · Notion |
| **B. API key 한 줄** | 사이트에서 키 발급 → `.env` 에 붙여넣기 | Firecrawl · YouTube · HeyGen · ElevenLabs · Buffer · Google Ads(토큰) |
| **C. OAuth / 자체 인증** | 자격증명 파일 또는 브라우저 인증 | Google Sheets · GA4 · Meta Ads(hosted) · Discord(Bot) |

### `.mcp.json` 의 역할

프로젝트 루트의 `.mcp.json` 에 적힌 MCP 는 **이 폴더에서 Claude 를 실행할 때 자동 로드**됩니다. 12개 골격은 이미 작성돼 있고, **영상에서 하는 일은 `.env` 에 키를 채우고 Claude.ai 통합을 켜는 것**뿐입니다.

> ⚠️ **반드시 marketing-os 루트에서 `claude` 실행** — 하위 폴더에서 실행하면 google-sheets 등 자체호스팅 MCP 가 로드되지 않습니다.

### 사전 준비 확인

영상 시작 전 `00-사전준비물.md` 의 체크리스트(파일 2개 + 키 8개 + Claude.ai 통합 3개)를 메모장 하나에 정리해두면 **영상 안 작업이 30~40분**에 끝납니다. 미완료면 사이트 이동 포함 75~90분.

# Ch 1. 데이터 분석 3개

## ① Google Sheets MCP — `01-google-sheets/` · [클립 1-2] · ⏱10분 · 패턴 C

- **한 줄**: 매출·광고·재고가 흩어진 시트 4개 → 자연어 한 줄로 종합 리포트. (절감 연 약 180시간)
- **서버**: 자체호스팅 `node mcp-server/index.js` · 도구 10개

**발급 (사전 준비)**
1. [Google Cloud Console](https://console.cloud.google.com) → 프로젝트 `marketing-os` 생성
2. **Google Sheets API** 사용 설정
3. OAuth 동의 화면(외부) + **테스트 사용자에 본인 이메일 추가** ⚠️
4. 사용자 인증 정보 → OAuth 클라이언트 ID → **애플리케이션 유형: 데스크톱 앱** ⚠️ → JSON 다운로드 → `oauth_credentials.json` 으로 이름 변경

**영상에서 Claude 에게**
```
~/Downloads/oauth_credentials.json 을 mcp-server/ 로 옮기고 google-sheets MCP 설치해줘
```
→ Claude: 파일 이동 + `cd mcp-server && npm install` + `"installed"` 키 검증

**헬스 체크** (재시작 후, 브라우저 OAuth 1회)
```
google-sheets 로 새 스프레드시트 하나 만들어줘
```
→ 시트 URL 반환되면 성공

> 🎬 **영상 멘트**: "데스크톱 앱 유형이 핵심입니다. `web` 으로 만들면 `installed` 키가 없어서 인증이 안 돼요."

## ② GA4 MCP — `02-ga4/` · [클립 1-3] · ⏱15분 · 패턴 C

- **한 줄**: GA4 화면 캡처·정리 주 1~2시간 → 자연어 1줄. (주간 리포트 60분 → 1분)
- **서버**: `npx -y mcp-server-ga4` · 핵심 도구 `runReport`

**발급 (사전 준비)** — 두 경로 중 하나
- **기본: 서비스 계정** — 같은 GCP 프로젝트에서 Google Analytics Data API 사용 → Service Account 생성 → JSON 키 다운로드 → analytics.google.com 에서 `client_email` 을 **뷰어 권한**으로 속성에 추가 → 속성 ID(9자리) 메모
- **대안: ADC** — Workspace 정책이 서비스 계정 키 발급을 막을 때(예: wmbb.kr). `gcloud auth application-default login` 으로 ADC 발급 후 `.mcp.json` 의 `GOOGLE_APPLICATION_CREDENTIALS` 가 이를 참조

**영상에서 Claude 에게**
```
GA4 속성 ID 는 427641480 이고, 서비스계정 JSON 은 ~/Downloads 에 있어. ga4 MCP 연결해줘
```

**헬스 체크**
```
ga4 로 지난 7일 활성 사용자 수 알려줘
```
→ 숫자 반환되면 성공

> ⚠️ **권한 부여 직후 PERMISSION_DENIED** 시 10분 대기 후 재시도. ADC 스코프 차단(invalid_rapt) 시 자체 OAuth 데스크톱 클라이언트로 ADC 재로그인 + Claude Code 재시작.
